一句话定位

SSC 分摊工作台是一个区域费用分摊自动化工具：运营上传台账 CSV/xlsx + 成本中心人数，系统自动按主体 join、按人数占比分摊、输出结果 CSV；同时内置按国家的 Special Case 处理规范中心，报错与文档一一配对，同事可自检自查。

台账来自财务导出的 xlsx/CSV，人数来自 HR 或手工维护的 CSV；工具做整合与计算，不另建独立数据源。
✅ 已做到 — ledger_config.json 外置映射，换表结构只改配置不改代码。
✅ 让用户在一个入口看到所有待办并驱动行动 PI

二、8 个必答自查问题

1. 本页面的目标用户角色是谁？
SSC 运营团队成员，按「正式人员 / 实习生」双 Tab 区分权限。26 名正式人员中 2 名 SUPER ADMIN（张文婧/魏梓棠），其余按国家分配操作/浏览权限。未选身份者被门禁拦截，无法使用任何功能。
2. 用户看完之后应该做什么操作？
① 选择身份 → 选择国家（US/CA）→ 上传台账 + 人数 CSV → 点击「执行分摊」→ 下载结果 CSV。② 如果出现报错 → 点击「查看处理指引」→ 跳转到对应国家的 Special Case 规范卡片 → 按「检查→修改→上传」四步修正 → 重传。
3. 用户操作的结果下游谁会消费？
分摊结果 CSV → 导入付款平台生成提单（下游：付款平台）。Special Case 报告 → 同事修正源数据 → 重传（闭环反馈）。日志记录 → 运营复盘 / 审计追踪。
4. 引用的数据是否符合 SSOT（唯一数据源）？
✅ 符合。台账来自财务系统导出（唯一 Owner）；人数来自 HR / 运营维护（唯一 Owner）；分摊结果由本工具产出（唯一 Owner，只读输出）。无跨系统双写。
5. 是否有纯展示、无交互的区块？
⚠️ 「协作」和「系统」面板当前为占位态（标注待上线），属灰度期过渡状态。计划下一迭代填入实际功能内容或合并移除。
6. 是否新增需要人工维护的数据源？成本谁来承担？
⚠️ 当前需手工维护「人数 CSV」（成本中心/部门/人数/主体四列），由各国运营自行维护。这是临时方案——后续接入 HR 数仓 API 后可自动拉取，消除人工维护成本。
7. 待办/告警是否接入统一收口？
📋 尚未接入。当前分摊结果和 warning 在页面内展示。规划接入总台统一待办列表（TAPD/飞书），让分摊任务和异常告警汇聚到一个入口。
8. UI 元素是否符合统一视觉规范？
✅ 已对齐 HR Dashboard Design Token（品牌色 #0052d9 / 功能色系 / 文字 4 级灰度 / 圆角 12px / 卡片阴影）。特殊场景：规范中心用了可展开卡片交互、结果页用了分级 badge（❌错误 ⚠️警告 ℹ️提示），均保持一致性。待 PMO + 设计师最终审核。

三、从运营反馈驱动的产品优化

反馈 1：「金额缺失/负数的行静默成功了，我没发现」
→ 加了数据质量报告（warnings），每类异常标明条数和处理方式；结果页顶部醒目展示。
反馈 2：「不同主体的费用串到一起分摊了」
→ 实现按「主体」join 分摊（CANADA→CA / BRL→BR / AMERICA→US），跨主体完全不串。
反馈 3：「人数表里同一个成本中心出现了两次，结果不对」
→ 成本中心合并逻辑修复：重复 CC 人数累加、部门取首个、冲突计数并 warning 提示。
反馈 4：「报错了不知道怎么修，要问人」
→ 内置 Special Case 规范中心（24 条，按国家分类），报错代码一键跳转对应文档，形成「报错→检查→修改→上传」闭环。
反馈 5：「没选身份的人也能操作，不安全」
→ 实现身份门禁：未选身份禁止切面板、导航标签锁定显示 🔒。
反馈 6：「上传了错误的表头格式，跑了半天才知道不对」
→ 加表头校验：缺必需列直接拒绝，精确告诉缺哪几列（如「人数表缺少：主体」）。

一套工具同时服务 美国(US) / 加拿大(CA) / 巴西(BR) 三国。按国家 tab 切换、按主体自动 join、按国别展示专属 Special Case 特例（如 US-TAX1 税务扣款 / CA-GST 消费税 / BR-CNPJ 税号校验）。

覆盖国家🇺🇸 US · 🇨🇦 CA · 🇧🇷 BR（+ 全球通用规范）
主体容错CANADA→CA · BRL→BR · America→US 自动规范化
国别特例US:2项 · CA:2项 · BR:2项（共 6 条国别 Special Case）

⏱️ 时长与优先级节约

新加坡案例：单次分摊从 ~40 分钟（Excel 手工 VLOOKUP + 交叉核对）降到 <3 分钟（上传→执行→下载）。

配合 Emily 解决的问题：台账格式不统一（xlsx 多 sheet / 列名差异）→ 实现 xlsx 自动提取 + 配置化映射（ledger_config.json），换表结构只改配置不改代码。

节约 ~90% 操作时间 消除人为复制粘贴错误

🤖 AI 幻觉防护 & 海外 Token 限制应对

AI 不替代判断，只做整理：xlsx 映射预览必须人工确认后才执行（不是全自动黑盒）。

Token 限制应对：分摊引擎是纯 Python 后端计算（不依赖 LLM token），仅在 xlsx 字段映射阶段可选性调用 AI 辅助识别。核心分摊逻辑零 token 消耗，可无限次运行。

幻觉防护：Special Case 规则硬编码在 JSON 中（非 AI 生成），每条规则有明确的触发条件/系统行为/修复动作/责任方。AI 输出始终附带置信度标注。

📋 Special Case 处理规范（自检自纠）

内置 24 条 Special Case，分为三级：

❌ 错误级（须修）S3 缺主体 / S4 无匹配 CC / H1-H6 人数侧异常 / bad_header 表头缺失
⚠️ 警告级（已处理）S1 金额缺失(按0) / S2 金额为负(正常分摊) / S6 疑似重复(标出不去重)
ℹ️ 提示级G3 四舍五入误差 / data_too_large 量过大提示

报错↔文档配对：结果页每条 error 带「查看处理指引」按钮，点击跳转到对应国家的规范卡片并展开。同事不需要问人，照着文档改就行。

🔐 权限配置体系

双 Tab 架构：正式人员（26 人）/ 实习生（4 人）分开管理。

四级权限矩阵：SUPER ADMIN（全权）→ 可操作（指定国家）→ 仅浏览 → 无权限。颜色编码一目了然（绿/蓝/灰）。

身份门禁：未选身份不能切面板、不能操作任何按钮。导航标签锁定显示 🔒。
